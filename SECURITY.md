# Security Policy

## Reporting a Vulnerability

If you discover a potential security issue in Orderly, please report it responsibly.

Please **do not publicly disclose security vulnerabilities with full exploit details** before maintainers have had time to investigate and address the issue.

Instead, please follow these guidelines.

---

## How to report

You can report a potential security issue by opening a GitHub issue with a brief description of the problem.

Please include:

- A short explanation of the issue
- Steps to reproduce (if applicable)
- The version of Orderly you are using
- Your operating system version

Avoid posting detailed exploit code publicly.

If necessary, maintainers may follow up privately for additional details.

---

## Scope

Security issues may include, but are not limited to:

- unintended file access
- privilege escalation
- unsafe file operations
- vulnerabilities affecting user data or system safety

---

## Responsible disclosure

We ask that security researchers allow time for investigation and fixes before publicly disclosing vulnerabilities.

This helps ensure that users remain protected.
