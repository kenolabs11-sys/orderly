# Contributing

Thank you for your interest in Orderly.

Orderly is primarily developed by a solo independent developer, but community feedback and suggestions are always welcome.

---

# Reporting bugs

If you encounter a bug, please open an issue.

Helpful information includes:

- Orderly version
- Windows version
- Steps to reproduce the problem
- Expected behavior
- Actual behavior
- Screenshots if applicable

If possible, include diagnostics logs generated by the application.

You can also submit a report directly from the app using:

Settings → Advanced → Diagnostics → **Send report**

---

# Feature requests

Feature ideas are welcome.

When suggesting a feature, please describe:

- the workflow you want to improve
- the problem you are trying to solve
- possible solutions or alternatives

Not all feature requests can be implemented, but feedback is valuable.

---

# Pull requests

At this time, external pull requests may not always be accepted.

Major changes should be discussed in an issue first.

---

# Code of conduct

Please be respectful and constructive when interacting in issues and discussions.
